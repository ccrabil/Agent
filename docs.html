Developers

Quickstart preview.

Three calls between your agent and its first verified outcome. Full documentation ships with pilot onboarding.

Step 1

Register the system.

Every agent gets an identity, an owner, and a policy scope.

POST /v1/systems

{
  "name": "support-agent-14",
  "owner": "cs-operations",
  "policy": "cs-default"
}
Step 2

Emit signals.

Actions, costs, and outcomes become sealed events.

POST /v1/signals

{
  "system": "support-agent-14",
  "type": "outcome.claimed",
  "action": "ticket.resolved",
  "cost_eur": 0.41,
  "value_eur": 18.50
}
Step 3

Get the proof.

Verified outcomes, ready for reports and billing.

GET /v1/reports/value?month=2026-03

{
  "outcomes_verified": 1203,
  "value_eur": 41200,
  "evidence": "sealed",
  "export": ["pdf", "invoice"]
}
Preview only. SDKs (Python, TypeScript), framework connectors, event streaming, and the full API reference are provided with pilot onboarding.
